# High-Converting Landing Page Layout (Design Only)

Use this skill when:
- Redesigning landing pages
- Structuring SaaS homepages
- Improving layout for conversion
- You already have content and copy

⚠️ DO NOT rewrite, improve, or change any copy.
Only restructure and design the page.

---

## 🎯 Primary Goal

Maximize conversions through:
- strong layout
- clear visual hierarchy
- clean spacing
- strategic CTA placement

---

## 📱 Mobile-First Requirement

Always design for mobile first:
- stack all sections vertically
- full-width elements
- large tap targets (buttons)
- readable font sizes
- minimal side-by-side layouts

Then scale up to desktop.

## 🧠 Core Layout Principles

### 1. Above-the-Fold Focus
Top section must include:
- headline
- supporting text
- primary CTA

Rules:
- keep it compact
- CTA visible without scrolling
- avoid clutter

---

### 2. One Goal Per Section
Each section should:
- communicate ONE idea
- support the main conversion

Avoid mixing:
- features + testimonials + CTAs in one block

---

### 3. Strong Visual Hierarchy

Order of attention:
1. headline
2. CTA
3. supporting text
4. visuals

Use:
- size
- spacing
- contrast

---

### 4. Section Flow (Required Structure)

Structure pages in this order:

#### 1. Hero
- headline
- subtext
- primary CTA

---

#### 2. Value / Benefits
- 3–5 key points
- use cards or stacked rows (mobile)

---

#### 3. Social Proof
- testimonials OR
- logos OR
- stats

---

#### 4. Features / How It Works
- simple breakdown
- use icons or visuals

---

#### 5. Secondary CTA
- reinforce action

---

#### 6. Final CTA Section
- repeat main CTA
- add urgency if present in content

---

## 🧩 Layout Rules

### Spacing
- use generous vertical spacing
- separate sections clearly
- avoid cramped layouts

---

### CTA Placement
- hero (required)
- mid-page (optional)
- final section (required)

CTA must:
- stand out visually
- be easy to tap on mobile

---

### Forms
- keep minimal
- single column
- avoid multi-field layouts on mobile

---

### Content Width
- mobile: full width with padding
- desktop: constrain to readable width (~max-w-5xl to 6xl)

---

### Grids
Mobile:
- 1 column

Desktop:
- 2–3 columns max

---

## 🎨 UI Design Rules

- minimalist design
- high contrast CTA
- consistent spacing
- clean typography
- avoid unnecessary colors

---

## 🚫 Do NOT Do

- do not rewrite text
- do not add new content
- do not create dense sections
- do not use complex layouts on mobile
- do not add multiple competing CTAs

---

## 🧪 Output Requirements

When using this skill:

1. First output:
   - page structure (sections + layout description)

2. Then output:
   - full UI code (Tailwind or similar)

3. Ensure:
   - mobile-first responsiveness
   - clean spacing
   - clear hierarchy

---

## 🔥 Summary

High-converting layout =

- clear structure
- strong hierarchy
- mobile-first simplicity
- obvious CTA placement

Your job is to:
👉 organize content into a layout that guides users to take action
👉 not to change what the content says
